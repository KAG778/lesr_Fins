# 需求：LESR 金融交易诊断与改进

**定义日期**：2026-04-14
**核心价值**：让 LESR 在金融数据上稳定优于纯 DQN 基线

## v1 需求

### 诊断工具（DIAG）

- [x] **DIAG-01**：系统可多次（≥10 次）独立运行 DQN baseline 和当前 LESR，收集统计样本
- [ ] **DIAG-02**：系统可自动对比 LESR vs DQN 的 Sharpe 比率差异，并计算统计显著性（t-test / bootstrap）
- [ ] **DIAG-03**：系统可分析每次 LLM 生成特征的质量（方差、与收益的相关性、信息比）
- [ ] **DIAG-04**：系统可分解不稳定性来源：LLM 采样波动 vs DQN 训练波动 vs 数据噪声
- [ ] **DIAG-05**：系统可记录每次运行的完整配置、LLM 输出代码、训练曲线、最终指标到结构化日志

### 训练/验证/测试重设计（EVAL）

- [ ] **EVAL-01**：系统支持 Walk-forward 滚动窗口训练/测试，而非仅固定窗口切分
- [ ] **EVAL-02**：系统支持多指标评估：Sharpe、Sortino ratio、最大回撤、Calmar ratio、胜率
- [ ] **EVAL-03**：系统防止 LLM 迭代优化中的数据泄露（LLM 反馈仅来自训练集分析）
- [ ] **EVAL-04**：系统支持按市场状态（牛市/熊市/震荡）分层评估策略表现
- [ ] **EVAL-05**：系统可汇总多股票、多窗口、多运行的结果并生成对比报告

### LESR 核心改进（LESR）

- [ ] **LESR-01**：Prompt 模板改进 — 更精确的金融语义引导，要求生成有经济学含义的特征
- [ ] **LESR-02**：LLM 输出质量守门 — 自动检查语法正确性、输出维度匹配、数值稳定性
- [ ] **LESR-03**：COT 反馈机制改进 — 仅反馈高置信度分析结果，包含"不要做什么"的负面反馈
- [ ] **LESR-04**：特征筛选机制 — 过滤退化特征，限制新增特征数量（5-10 个）
- [ ] **LESR-05**：特征稳定性评估 — 测试同一特征在不同子期间的一致性

## v2 需求（延后）

### 高级特性

- **ADVF-01**：自适应特征库 — 维护已验证有效的特征库，新迭代在此基础上增量优化
- **ADVF-02**：市场状态感知 prompt — 根据当前股票特性（波动率、趋势）动态调整 prompt
- **ADVF-03**：集成特征选择 — 从多次 LLM 采样中集成最优特征子集
- **ADVF-04**：Combinatorial Purged Cross-validation（CPCV）

## 超出范围

| 特性 | 原因 |
|------|------|
| 自定义 LLM 训练 | 成本和复杂度过高，使用现有 API |
| 替换 DQN 为其他 RL 算法 | 先聚焦 LESR 层面，除非诊断表明 DQN 是瓶颈 |
| 实时交易系统 | 研究阶段不需要 |
| Web 界面 | 非研究核心 |
| Stable-Baselines3/Ray | 当前 DQN 实现够用，替换成本高 |
