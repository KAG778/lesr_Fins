# LESR 金融交易 — 诊断与改进

## 项目描述

LESR（LLM-Empowered State Representation）是一个利用 LLM 为强化学习（DQN）生成特征工程函数的研究框架。当前核心问题：LESR 生成的状态表示相比纯 DQN 基线在 Sharpe 比率上表现不稳定，无法可靠地优于基线。项目目标是系统诊断 LESR 不稳定的根因，修复设计缺陷，并重新设计训练/验证/测试流程，使 LESR 稳定优于纯 DQN。

## 核心价值

**让 LESR 在金融数据上稳定优于纯 DQN 基线** — 如果 LLM 生成的特征不能可靠地比原始特征更好，整个方法的价值就无法成立。

### 待实现

- [ ] LESR 不稳定性根因的系统诊断
- [ ] LLM prompt 和反馈机制的改进
- [ ] 迭代优化循环设计的修复
- [ ] 训练/验证/测试流程的重新设计
- [ ] 可靠的实验评估体系（统计显著性检验）

- 实时交易系统 — 研究阶段不需要
- Web 界面或可视化工具 — 非研究核心

## 背景

### 当前问题

LESR 在金融数据上表现不稳定：
1. **同一配置多次运行结果波动大** — LLM 生成的代码质量不一致
2. **不同股票/时间窗口间差异大** — 特征生成缺乏适应性
3. **训练/验证/测试流程设计不合理** — 固定时间窗口切分可能不够稳健

### 可能的根因方向

- LLM prompt 设计问题（特征引导不够精确）
- COT 反馈机制无效（分析结果未能有效引导 LLM 改进）
- 原始 LESR 的 Lipschitz 分析不适用金融数据（已替换为 SHAP，但替代方案是否足够？）
- 特征选择/筛选机制不够严格
- 内在奖励信号设计问题
- 训练评估流程存在数据泄露或过拟合

### 技术环境

- Python 3.8+，PyTorch，OpenAI API (GPT-4o-mini via ChatAnywhere)
- FINSABER 回测框架，FinMem 数据格式
- 4 只股票（TSLA/MSFT/AMZN/NFLX），多时间窗口（2012-2024）

## 约束

- **方法论**：必须保留 LLM 生成特征工程函数的核心思路
- **数据与框架**：复用现有 FINSABER 回测框架和 FinMem 数据
- **发表要求**：最终方案需要可写成论文发表
- **资源**：使用现有 LLM API 和 GPU 资源

## 关键决策

| 决策 | 理由 | 结果 |
|------|------|------|
| 特征分析用 SHAP 替代 Lipschitz | 金融数据不满足 Lipschitz 假设（非连续、高噪声、非平稳） | — 待验证 |
| 使用 GPT-4o-mini 作为 LLM | 成本与质量平衡 | — 待评估 |
