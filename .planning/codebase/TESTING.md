# 测试

## 测试现状

**项目没有系统化的测试套件。** 验证主要依赖：
- 手动运行实验脚本
- 观察日志输出和结果指标
- 与基线策略对比

## 存在的测试文件

- `exp4.7/test_baseline_import.py` — 验证基线模块能否正常导入
- `exp4.7/test_data.py` — 验证数据加载是否正常
- `llm_rl_trading_finsaber/tests/` — 重构版本有测试目录（内容待确认）

## 验证方式

### 实验验证
- 每次实验运行后自动保存结果到 `results_*/` 或 `result_*/` 目录
- 结果包含：训练日志、验证指标（夏普比率、收益率、最大回撤）
- 每个样本的 DQN 训练产出 `episode_rewards`、`episode_states`

### 基线对比
- `baseline.py` 提供无特征工程的 DQN 基线
- `train_baseline_dqn()` 函数训练并评估基线策略
- 对比指标：夏普比率、累计收益率、最大回撤

### 特征分析验证
- `feature_analyzer.py` 使用 Spearman 相关性 + SHAP 值分析特征贡献
- 验证 LLM 生成的特征是否对策略有帮助

## 缺失的测试

- **单元测试**：核心模块（DQN、特征分析、prompt 生成）无单元测试
- **集成测试**：LLM → 代码解析 → DQN 训练 → 评估的端到端测试
- **回归测试**：配置变更后的回归检测
- **数据验证**：数据加载后的完整性检查
- **LLM 输出验证**：生成的代码是否能正确执行、输出维度是否正确

## CI/CD

- 无 CI/CD 管道
- 无自动化测试运行
- 完全依赖手动运行验证
