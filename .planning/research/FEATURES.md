# 特性研究

## 必备特性（Table Stakes）

### 诊断工具
1. **实验追踪与对比** — 记录每次运行的配置、LLM 输出、训练曲线、最终指标
2. **统计显著性检验** — 多次运行对比 LESR vs DQN，用 t-test 或 bootstrap 确认差异
3. **特征质量诊断** — 分析 LLM 生成的特征（方差、相关性、信息量），识别低质量特征
4. **训练曲线可视化** — 观察 DQN 是否收敛、是否过拟合

### 评估体系
5. **多指标评估** — 除了 Sharpe，还需 Sortino ratio、最大回撤、Calmar ratio、胜率
6. **Walk-forward 验证** — 滚动训练/测试，而非固定切分
7. **多股票多窗口聚合** — 汇总所有测试结果，而非看单个案例
8. **基线对比框架** — 与 Buy&Hold、MA交叉等经典策略对比，不只是 DQN

### LLM 优化循环
9. **Prompt 模板版本管理** — 跟踪 prompt 变化对结果的影响
10. **LLM 输出质量筛选** — 过滤语法错误、维度错误、退化特征的代码

## 差异化特性（Differentiators）

11. **自适应特征库** — 维护一个已验证有效的特征库，新迭代在此基础上增量优化
12. **股票/市场状态感知 prompt** — 根据当前股票特性（波动率、趋势）动态调整 prompt
13. **特征稳定性评分** — 评估特征在不同时间段的一致性
14. **集成特征选择** — 从多次 LLM 采样中集成最优特征子集

## 反特性（Anti-features — 不应该做的）

1. **不要让 LLM 直接输出交易信号** — LLM 的价值在于特征工程，不是决策
2. **不要追求单次运行最优** — 目标是统计意义上的稳定优势
3. **不要忽略交易成本** — 回测必须包含佣金和滑点
4. **不要过度增加特征维度** — 过多特征导致过拟合，控制在 5-15 个新增特征
