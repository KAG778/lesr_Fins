# 陷阱研究

## 陷阱 1：LLM 特征生成的随机性陷阱

**描述**：LLM 每次调用生成的特征函数不同，导致结果本质上就是高方差的。这不是 bug，而是 LLM 作为优化器的固有特性。

**预警信号**：
- 同一 prompt 多次调用生成完全不同的特征
- 不同特征在某些窗口偶然表现好

**预防策略**：
- 多次采样取最优（当前已做，sample_count=6）
- 但需增加统计评估 — 最优样本是否优于 DQN baseline 的平均表现？
- 考虑特征一致性筛选 — 只保留多次采样中都出现的特征类型

**应处理阶段**：阶段 1（诊断）和阶段 2（修复）

## 陷阱 2：Sharpe 比率幻觉

**描述**：单次运行的 Sharpe 比率差异没有统计意义。金融时间序列自相关强，有效样本量远小于表面样本量。

**预警信号**：
- Sharpe 差异 < 0.5 就宣称"优于"
- 只展示最优结果，忽略其他样本

**预防策略**：
- 使用 Deflated Sharpe Ratio（考虑多重测试的修正）
- Bootstrap 置信区间
- 最低要求：至少 10 次独立运行的统计检验

**应处理阶段**：阶段 1（诊断）

## 陷阱 3：数据泄露 — 未来信息渗入训练

**描述**：LLM 迭代优化中使用验证集信息选择最优样本，实质上是对验证集的过拟合。

**预警信号**：
- 验证集表现持续提升但测试集没有
- 迭代次数越多，测试集优势反而越小

**预防策略**：
- 严格的 train → validation → test 分层
- LLM 反馈仅来自训练集的分析
- 最终评估只在测试集上进行一次
- 考虑 combinatorial purged cross-validation

**应处理阶段**：阶段 2（训练流程重设计）

## 陷阱 4：特征过拟合 — 金融数据噪声比信号强

**描述**：LLM 生成的特征可能在训练集上看似有效（偶然相关性），但在新数据上失效。金融数据信噪比极低（SNR ≈ 5-10dB）。

**预警信号**：
- 特征重要性分析显示所有特征都"重要"（可能是噪声）
- 训练 Sharpe 远高于测试 Sharpe

**预防策略**：
- 要求 LLM 生成有经济学含义的特征（而非纯统计特征）
- 特征必须通过 out-of-sample 验证
- 限制新增特征数量（5-10 个而非 30+）
- 特征稳定性测试 — 同一特征在不同子期间的相关性

**应处理阶段**：阶段 2（Prompt 和筛选改进）

## 陷阱 5：COT 反馈噪声 — 无意义的反馈反而误导 LLM

**描述**：当前的 COT 反馈可能包含噪声信息（如偶然相关的特征被标记为"重要"），反而误导下一轮 LLM 生成更差的代码。

**预警信号**：
- 迭代性能没有单调提升趋势
- 后期迭代的表现比早期差

**预防策略**：
- 只反馈高置信度的分析结果
- 反馈中包含"不要做什么"（负面反馈比正面反馈更有效）
- 考虑 ablation — 对比有/无 COT 反馈的 LESR

**应处理阶段**：阶段 2（反馈循环改进）

## 陷阱 6：固定窗口评估 — 忽略市场状态变化

**描述**：固定时间窗口的 train/val/test 切分可能恰好包含了特定市场状态（牛市/熊市/震荡），结果不具有泛化性。

**预警信号**：
- 不同窗口（W1-W10）的结果差异巨大
- 训练期和测试期的市场状态不同

**预防策略**：
- Walk-forward 分析 — 多个滚动窗口
- 按市场状态分层评估（牛市/熊市/震荡分别报告）
- 确保训练和测试数据覆盖不同市场状态

**应处理阶段**：阶段 3（验证）

## 陷阱 7：Lipschitz 假设不成立（已识别但替代方案可能不够）

**描述**：原始 LESR 论文基于 Lipschitz 连续性假设，但金融数据有跳空、限价、非平稳等特性，不满足该假设。项目已用 SHAP 替代，但 SHAP 是否足够？

**预警信号**：
- SHAP 值在不同子期间剧烈变化
- 特征重要性排名不稳定

**预防策略**：
- 使用多种特征评估方法交叉验证
- 关注特征的时间一致性而非静态重要性
- 考虑条件特征重要性（在不同市场状态下分别评估）

**应处理阶段**：阶段 2（特征分析增强）
