# 研究总结

## 核心发现

### 技术栈
- 当前技术栈（PyTorch + OpenAI API + SHAP + FINSABER）基本适配，不需要大规模替换
- 瓶颈不在工具，在于方法论：prompt 设计、评估方法、反馈循环
- 建议新增：统计检验（scipy.stats）、Walk-forward 框架、结构化结果存储

### 关键特性
- **必备**：实验追踪、统计显著性检验、多指标评估、Walk-forward 验证、LLM 输出质量筛选
- **差异化**：自适应特征库、市场状态感知 prompt、特征稳定性评分、集成特征选择
- **反特性**：LLM 不应直接输出交易信号、不要追求单次最优

### 架构
- 推荐 **诊断 → 修复 → 验证** 三阶段架构
- 组件边界：诊断模块（新增）→ LESR 核心改进（修改）→ 评估框架（新增）→ 训练管道（重设计）
- 构建顺序：先诊断理解问题 → 再修复基础设施 → 然后改进核心 → 最后全面验证

### 陷阱
- **最大风险**：LLM 特征随机性 + Sharpe 幻觉 — 可能把噪声当成信号
- **数据泄露**：LLM 迭代优化中验证集信息可能渗入训练
- **COT 反馈噪声**：无意义的反馈可能误导 LLM 生成更差的代码
- **特征过拟合**：金融数据 SNR 极低，特征容易虚假相关
