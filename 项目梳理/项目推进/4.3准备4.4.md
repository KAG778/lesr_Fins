### 讨论要点

#### 2.1 数值→语义的映射层次

```

├─ 成交量: 50,000,000
└─ 问题: 无明确含义

├─ 风险: "波动率偏高，注意风险"
└─ 信号: "成交量放大，趋势确认"
```

#### 2.2 典型特征分类（先跑通链路）

**第一类: 趋势类特征（必须）**
```python
# 1. 多时间尺度动量
momentum_5d = (close[-1] - close[-6]) / close[-6]
momentum_10d = (close[-1] - close[-11]) / close[-11]
momentum_20d = (close[-1] - close[-21]) / close[-21]

# 2. 移动平均线趋势
ma_trend = (sma5 - sma20) / sma20  # 正值=上升趋势

# 3. 趋势强度
trend_strength = abs(sma5 - sma20) / close[-1]
```

**第二类: 风险类特征（必须）**
```python
# 1. 历史波动率
volatility = np.std(returns[-20:])

# 2. ATR (真实波幅)
atr = average_true_range(high, low, close, 14)

# 3. 下行风险
downside_risk = np.std(returns[returns < 0])
```

**第三类: 动量类特征（推荐）**
```python
# 1. RSI (相对强弱指标)
rsi = compute_rsi(close, 14)

# 2. MACD
macd = ema(close, 12) - ema(close, 26)

# 3. 随机指标
stoch_k = (close - low_14) / (high_14 - low_14) * 100
```

**第四类: 成交量类特征（推荐）**
```python
# 1. 成交量变化率
volume_change = (volume[-1] - volume[-6]) / volume[-6]

# 2. 价量关系
price_volume_trend = price_change * (volume[-1] / volume_ma20)

# 3. 成交量相对强度
volume_ratio = volume[-1] / volume_ma20
```

#### 2.3 特征精简建议（MVP）

**最小可行特征集（8个）**:
```python
def revise_state_mvp(raw_state):
    """
    MVP版本：8个核心特征
    """
    close = raw_state[0:20]
    volume = raw_state[80:100]

    # 趋势 (3个)
    mom_5 = (close[-1] - close[-6]) / close[-6]
    mom_10 = (close[-1] - close[-11]) / close[-11]
    ma_trend = (np.mean(close[-5:]) - np.mean(close[-20:])) / close[-1]

    # 风险 (2个)
    returns = np.diff(close) / close[:-1]
    volatility = np.std(returns[-20:])
    downside_vol = np.std(returns[returns < 0])

    # 动量 (1个)
    rsi = compute_rsi(close, 14)

    # 成交量 (2个)
    vol_change = (volume[-1] - volume[-6]) / volume[-6]
    vol_ma_ratio = volume[-1] / np.mean(volume[-20:])

    return np.concatenate([
        raw_state,
        [mom_5, mom_10, ma_trend, volatility, downside_vol,
         rsi, vol_change, vol_ma_ratio]
    ])
```

## 问题3: Lipschitz替代方案与评测

### 背景分析

**核心问题**:
- LESR使用Lipschitz常数评估特征重要性
- Lipschitz假设：状态-奖励映射连续
- 金融数据违反所有假设（跳变、噪声、非平稳）

### 讨论要点

#### 3.1 Lipschitz失效的根本原因

| Lipschitz假设 | 金融数据现实 | 结果 |
|--------------|-------------|------|
| 状态空间连续 | 价格跳变、涨跌停 | ❌ 违反 |
| 奖励函数光滑 | 盈利瞬间跳变 | ❌ 违反 |
| 低噪声 | 高噪声（SNR≈8dB） | ❌ 违反 |
| 平稳过程 | 非平稳（分布漂移） | ❌ 违反 |
| 特征独立 | 高度共线性 | ❌ 违反 |

#### 3.2 替代方案对比

| 方案 | 适用性 | 复杂度 | 可解释性 | 计算成本 | 推荐度 |
|------|--------|--------|----------|----------|--------|
| **相关性分析** | ⭐⭐⭐⭐ | 低 | 高 | 低 | ⭐⭐⭐⭐⭐ |
| **SHAP值** | ⭐⭐⭐⭐⭐ | 中 | 中 | 中 | ⭐⭐⭐⭐ |
| **排列重要性** | ⭐⭐⭐⭐ | 低 | 高 | 高 | ⭐⭐⭐ |
| **互信息** | ⭐⭐⭐ | 中 | 低 | 中 | ⭐⭐ |
| **因果推断** | ⭐⭐⭐ | 高 | 高 | 高 | ⭐⭐⭐ |

#### 3.3 推荐方案: 分层实施

**阶段1: 相关性分析（立即实施）**
```python
def correlation_analysis(features, rewards, feature_names):
    """
    替代Lipschitz的基础方案

    优势:
    - 计算快速 O(n)
    - 直观易懂
    - 适合线性关系

    输出: 每个特征与收益的相关系数
    """
    from scipy import stats

    results = {}
    for i, name in enumerate(feature_names):
        # Pearson相关系数
        pearson_r, pearson_p = stats.pearsonr(features[:, i], rewards)

        # Spearman秩相关（更鲁棒）
        spearman_r, spearman_p = stats.spearmanr(features[:, i], rewards)

        results[name] = {
            'pearson': pearson_r,
            'pearson_p': pearson_p,
            'spearman': spearman_r,
            'spearman_p': spearman_p,
            'importance': abs(spearman_r)  # 综合重要性
        }

    return results
```

**阶段2: SHAP值分析（1-2周后）**
```python
def shap_analysis(model, features, feature_names):
    """
    增强方案：捕获非线性关系

    优势:
    - 理论基础扎实（Shapley值）
    - 捕获特征交互
    - 模型无关

    输出: 每个特征的SHAP重要性
    """
    import shap

    explainer = shap.Explainer(model, features)
    shap_values = explainer(features)

    importance = np.abs(shap_values.values).mean(axis=0)

    return dict(zip(feature_names, importance))
```

**阶段3: 组合评估（稳定后）**
```python
def combined_importance(features, rewards, model, feature_names):
    """
    组合多种方法，给出更稳健的评估

    方法:
    1. 相关性分析（线性）
    2. SHAP值（非线性）
    3. 排列重要性（模型无关）

    输出: 综合重要性评分
    """
    # 方法1: 相关性
    corr_imp = correlation_analysis(features, rewards, feature_names)

    # 方法2: SHAP
    shap_imp = shap_analysis(model, features, feature_names)

    # 方法3: 排列重要性
    perm_imp = permutation_importance(model, features, rewards)

    # 综合评分（加权平均）
    combined = {}
    for name in feature_names:
        combined[name] = (
            0.3 * normalize(corr_imp[name]['importance']) +
            0.5 * normalize(shap_imp[name]) +
            0.2 * normalize(perm_imp[name])
        )

    return combined
```

#### 3.4 评测方案

**评测维度**:
```
1. 预测力 (Predictive Power)
   ├─ 与未来收益的相关性
   ├─ 方向预测准确率
   └─ 信息比率 (Information Ratio)

2. 稳定性 (Stability)
   ├─ 时间稳定性（不同时期）
   ├─ 股票稳定性（不同标的）
   └─ 市场环境稳定性（牛熊震荡）

3. 独立性 (Independence)
   ├─ 特征间相关性（避免冗余）
   ├─ VIF (方差膨胀因子)
   └─ 条件数分析

4. 可解释性 (Interpretability)
   ├─ 金融含义清晰
   ├─ 与理论一致
   └─ 直观易懂
```

**评分公式**:
```python
def score_feature(feature_name, metrics):
    """
    综合评分

    Score = 0.4 × 预测力 + 0.3 × 稳定性 + 0.2 × 独立性 + 0.1 × 可解释性
    """
    score = (
        0.4 * metrics['predictive_power'] +
        0.3 * metrics['stability'] +
        0.2 * metrics['independence'] +
        0.1 * metrics['interpretability']
    )
    return score
```

#### 4.3 提示词模板设计

**初始化模板关键组件**:
```python
FINSABER_INIT_PROMPT = """
你是金融交易领域的量化分析专家。

## 任务背景

使用强化学习训练股票交易策略。策略在每个交易日决定：
- 买入 (Buy): 建立多头仓位
- 卖出 (Sell): 平仓或建立空头仓位
- 持有 (Hold): 保持当前仓位

## 可用数据

原始状态 (120维):
- s[0:19]: 20天收盘价序列
- s[20:39]: 20天开盘价序列
- s[40:59]: 20天最高价序列
- s[60:79]: 20天最低价序列
- s[80:99]: 20天成交量序列
- s[100:119]: 20天调整后收盘价序列

## 金融语义说明

**关键概念**:
1. **收益率 (Return)**: 价格变化的百分比
   - 计算公式: (P_t - P_{t-1}) / P_{t-1}
   - 金融含义: 投资收益的度量

2. **波动率 (Volatility)**: 价格变动的剧烈程度
   - 计算公式: 收益率的标准差
   - 金融含义: 风险的度量

3. **趋势 (Trend)**: 价格的持续运动方向
   - 上升趋势: 价格持续上涨
   - 下降趋势: 价格持续下跌
   - 震荡趋势: 价格横盘整理

4. **动量 (Momentum)**: 价格变化的速度
   - 正动量: 上涨加速
   - 负动量: 下跌加速

5. **支撑/阻力 (Support/Resistance)**: 价格难以突破的水平
   - 支撑位: 价格下跌时的"地板"
   - 阻力位: 价格上涨时的"天花板"

## 目标函数

策略的绩效由以下指标衡量:
1. **夏普比率 (Sharpe Ratio)**: 风险调整后收益
   - 公式: (收益率 - 无风险利率) / 波动率
   - 目标: 最大化夏普比率

2. **最大回撤 (Max Drawdown)**: 最大损失幅度
   - 公式: max((峰值 - 当前值) / 峰值)
   - 目标: 最小化最大回撤

3. **总收益 (Total Return)**: 累积收益率
   - 目标: 最大化总收益

## 约束条件

1. **交易成本**: 每笔交易收取0.1%佣金
2. **仓位限制**: 单只股票最大仓位100%
3. **风险限制**: 单日最大亏损不超过5%

## 任务要求

请生成两个函数:

### 函数1: revise_state(raw_state)
输入: 原始状态 (120维)
```

## 问题5: 数据流与架构框架核对

### 背景分析

**核心理解**:
- LESR: 物理仿真环境（固定、确定）
- FINSABER: 金融交易环境（变化、随机）
- 需要核对数据流和适配架构

### 讨论要点

#### 5.1 LESR数据流（参考）

```
┌─────────────────────────────────────────────────────────┐
│                    LESR数据流                            │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  环境状态 (s_t)                                          │
│      ↓                                                  │
│  revise_state(s_t) ───→ 增强状态 (s'_t)                 │
│      ↓                                                  │
│  Actor网络(s'_t) ───→ 动作 (a_t)                        │
│      ↓                                                  │
│  MuJoCo环境(a_t) ───→ 下一状态 (s_{t+1})               │
│      ↓                                                  │
│  奖励 = r_env + w × intrinsic_reward(s'_t)              │
│      ↓                                                  │
│  存储到ReplayBuffer                                     │
│      ↓                                                  │
│  Episode结束 → Lipschitz分析 → COT反馈                  │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

#### 5.2 FINSABER数据流（适配后）

```
┌─────────────────────────────────────────────────────────┐
│                 FINSABER数据流（适配版）                   │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  市场数据 (价格序列)                                     │
│      ↓                                                  │
│  数据预处理 → 原始状态 (s_t)                             │
│      ↓                                                  │
│  revise_state(s_t) ───→ 增强状态 (s'_t)                 │
│      ↓                                                  │
│  LLM/Actor网络(s'_t) ───→ 交易信号 (buy/sell/hold)      │
│      ↓                                                  │
│  FINSABER回测引擎 ───→ 下一状态 (s_{t+1})               │
│      ↓                                                  │
│  奖励 = 收益 + w × intrinsic_reward(s'_t)               │
│      ↓                                                  │
│  存储到经验池                                           │
│      ↓                                                  │
│  Window结束 → 相关性/SHAP分析 → COT反馈                  │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

#### 5.3 架构差异与适配

| 组件 | LESR | FINSABER | 适配方案 |
|------|------|----------|----------|
| **环境** | MuJoCo | FINSABER回测 | ✅ 已有 |
| **状态表示** | revise_state | 需要适配 | 🔧 待实现 |
| **策略** | TD3 Actor | LLM决策 | 🔧 待实现 |
| **奖励** | 环境+内在 | 收益+内在 | 🔧 待实现 |
| **分析** | Lipschitz | 相关性/SHAP | 🔧 待实现 |
| **反馈** | COT | 金融COT | 🔧 待实现 |
