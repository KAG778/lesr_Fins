## 3. 状态契约对比图

### 3.1 Generic vs Native 对比

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    Generic Schema vs Native Contract                        │
└─────────────────────────────────────────────────────────────────────────────┘

Generic LESR Schema (原始机器人控制)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
维度公式: len(global) + len(assets) × (6 + len(indicators))

字段顺序 (Asset-Major):
┌─────────────────────────────────────────────────────────────────────────────┐
│ Global Features (2 dims)                                                 │
│ • s[0]: cash                                                            │
│ • s[1]: portfolio_value                                                 │
├─────────────────────────────────────────────────────────────────────────────┤
│ Per-Asset Features (6 + len(indicators) dims, repeated for each asset)    │
│ • s[2:8]:   Asset 0 - [open, high, low, close, volume, holding]          │
│ • s[8:14]:  Asset 1 - [open, high, low, close, volume, holding]          │
│ • s[14:20]: Asset 2 - [open, high, low, close, volume, holding]          │
│ • ...                                                                   │
│ • s[...]:  Indicators (SMA, RSI, ...) - asset-major order               │
└─────────────────────────────────────────────────────────────────────────────┘

FINSABER Native Contract (当前项目)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
维度公式: 1 + 2×stock_dim + len(indicators)×stock_dim

字段顺序 (Indicator-Major for indicators):
┌─────────────────────────────────────────────────────────────────────────────┐
│ Cash (1 dim)                                                             │
│ • s[0]: cash                                                            │
├─────────────────────────────────────────────────────────────────────────────┤
│ Close Prices (stock_dim dims)                                            │
│ • s[1:6]:   [close_0, close_1, close_2, close_3, close_4]                │
├─────────────────────────────────────────────────────────────────────────────┤
│ Holdings (stock_dim dims)                                                │
│ • s[6:11]:  [holding_0, holding_1, holding_2, holding_3, holding_4]      │
├─────────────────────────────────────────────────────────────────────────────┤
│ Indicators (stock_dim × len(indicators) dims, Indicator-Major)           │
│ • s[11:16]: [SMA_0, SMA_1, SMA_2, SMA_3, SMA_4]   ← SMA block           │
│ • s[16:21]: [RSI_0, RSI_1, RSI_2, RSI_3, RSI_4]   ← RSI block           │
│ • s[21:26]: [VOL_0, VOL_1, VOL_2, VOL_3, VOL_4]   ← Volatility block    │
└─────────────────────────────────────────────────────────────────────────────┘

⚠️  关键差异:
━━━━━━━━━━━━━━
1. ❌ Native 没有 open/high/low/volume
2. ❌ Native 的 indicators 按 indicator-major 排列
3. ❌ Native 的 close 和 holdings 是独立的 block
4. ⚠️  Prompt 必须明确说明这些差异
```

### 3.2 索引访问示例

```
┌─────────────────────────────────────────────────────────────────────────────┐
```
