### 1.2 定制策略矩阵

| 任务类型 | 状态空间特征 | 关键物理量 | 领域提示 | 性能指标 |
|---------|-------------|-----------|---------|---------|
| **运动任务** (HalfCheetah) | 关节角度 + 角速度 | 前向速度、能量、稳定性 | 不需要或简单 | 累积奖励 |
| **导航任务** (AntMaze) | 位置 + 关节状态 + 目标 | 距离、方向、身体协调 | 必需 (分阶段学习) | 成功率 |
| **操作任务** (Fetch) | 位置 + 关节 + 物体 | 抓取力、物体位置、距离 | 必需 (先抓取再放置) | 成功率 |
| **平衡任务** (Hopper) | 关节角度 + 角速度 | 高度、角度、能量 | 简单 (保持直立) | 累积奖励 |

开始
  │

  │   ├─ 最大化速度/距离 → 运动任务 → 转到 Step A
  │   ├─ 到达目标位置 → 导航任务 → 转到 Step B
  │   └─ 操作物体 → 操作任务 → 转到 Step C
  │

  │   ├─ 关键维度: 前向速度维度

  │   ├─ 特征建议: 速度、能量、协调性
  │   └─ 性能指标: Cumulative Reward
  │
  ├─ Step B: 导航任务定制
  │   ├─ 状态空间: 位置 (m) + 关节 + 目标 (m)
  │   ├─ 关键维度: 目标距离、方向向量
  │   ├─ 领域提示: 必需！"先学走路，再学走向目标"
  │   ├─ 特征建议: 距离、方向、身体稳定性
  │   └─ 性能指标: Success Rate
  │
  └─ Step C: 操作任务定制
      ├─ 状态空间: 位置 + 关节 + 物体姿态
      ├─ 关键维度: 物体距离、抓取姿态
      ├─ 领域提示: 必需！"先学抓取，再学放置"
      ├─ 特征建议: 抓取稳定性、物体位置、接近度
      └─ 性能指标: Success Rate
```

### 2.4 预期 LLM 输出分析

```python
# ========== 预期的成功案例 ==========
"""
LLM 应该生成的特征:

1. 前向速度特征 (必需)
   - forward_velocity = s[8] (x 方向速度)
   - 为什么？这是任务的核心目标

2. 能量相关特征 (推荐)
   - kinetic_energy = 0.5 * sum(s[10:17]**2)  (动能)
   - energy_consumption = sum(abs(s[10:17] * s[3:10]))  (功率)
   - 为什么？奖励应该考虑能量效率

3. 稳定性特征 (推荐)
   - body_height = s[0]  (身体高度)
   - height_stability = -abs(s[0] - target_height)
   - 为什么？保持稳定的高度有助于持续运动

4. 协调性特征 (可选)
   - coordination = std(s[10:17])  (运动平滑度)
   - front_back_balance = mean(s[10:13]) - mean(s[14:17])
   - 为什么？协调的运动更高效

预期的 intrinsic_reward 设计:
   reward = w1 * forward_velocity - w2 * energy_consumption + w3 * stability
   其中 w1 > w2 > w3 (速度是主要目标)
"""

# ========== 实际 LLM 输出示例 (第 1 轮) ==========
"""
```python
import numpy as np

def revise_state(s):
    # Extract joint positions and velocities
    positions = s[0:9]
    velocities = s[9:17]

    # Feature 1: Forward velocity (primary objective)
    forward_velocity = s[8]

    # Feature 2: Energy consumption
    energy = np.sum(np.abs(positions * velocities))

    # Feature 3: Body height (stability)
    body_height = s[0]

    # Feature 4: Movement smoothness
    smoothness = -np.std(velocities)

    return np.concatenate([s, [forward_velocity, energy, body_height, smoothness]])

def intrinsic_reward(updated_s):
    forward_velocity = updated_s[27]
    energy = updated_s[28]
    smoothness = updated_s[30]

    # Reward forward progress, penalize energy, reward smoothness
    reward = 10.0 * forward_velocity - 0.5 * energy + 2.0 * smoothness
    return np.clip(reward, -100, 100)
```

分析:
✅ 正确识别了前向速度 (s[8]) 作为关键特征
✅ 添加了能量消耗特征，考虑效率
✅ 使用了平滑度特征，鼓励协调运动
✅ intrinsic_reward 使用了添加的维度
✅ 使用了 clip 确保数值稳定性

性能: 最终奖励 4500.23 (超过 baseline 35%)
"""

## 三、实战案例 2：AntMaze（导航任务）

### 3.1 任务分析

```python
# ========== 任务基本信息 ==========
任务名称: AntMaze-v4
任务类型: 连续控制 - 导航任务
状态维度: 27
动作维度: 8
目标: 从起点到达指定目标位置

# ========== 状态空间分解 ==========
"""
维度分布:
├─ s[0:2] (2 维): 身体位置 (x, y)
│   ├─ s[0]: x 坐标
│   ├─ s[1]: y 坐标
│   └─ 单位: m (米)
│
├─ s[2] (1 维): 身体高度 (z)
│   └─ 单位: m (米) ← 稳定性指标
│
├─ s[3:15] (12 维): 4 条腿的关节角度
│   ├─ 每条腿 3 个关节
│   └─ 单位: rad (弧度)
│
└─ s[15:27] (12 维): 4 条腿的关节角速度
    └─ 单位: rad/s (弧度/秒)

关键维度识别:
✅ s[0:2] = 身体位置 (x, y) ← 最重要！决定是否到达目标
✅ s[2] = 身体高度 (z) ← 稳定性指标
✅ s[3:15] = 腿部关节角度 ← 协调性关键
✅ s[15:27] = 腿部关节角速度 ← 运动模式

目标信息 (通常在额外维度):
- goal_x, goal_y: 目标位置
- distance_to_goal: 距离目标的距离
- angle_to_goal: 相对目标的角度
"""

# ========== 物理模型理解 ==========
"""
AntMaze 的物理约束:
1. 身体结构: 3D 四足机器人，每条腿 3 个关节
2. 自由度: 12 个可驱动关节 (4 腿 × 3 关节)
3. 运动限制: 在迷宫中移动，不能穿墙
4. 奖励机制:
   - 成功奖励: 到达目标位置 (±0.5m)
   - 距离惩罚: 与目标距离
   - 碰撞惩罚: 撞墙

关键物理量:
- 位置 (x, y): 当前位置
- 距离 (d) = sqrt((x - goal_x)^2 + (y - goal_y)^2)
- 方向 (θ) = atan2(goal_y - y, goal_x - x)
- 身体稳定性: z 轴高度变化

四足协调模式:
- 对角线步态: 左前 + 右后 同步，右前 + 左后 同步
- 这是四足动物的自然步态
"""

# ========== 任务描述撰写 ==========
task_description = """
The agent is an ant-like robot with 4 legs, each leg has 3 joints. The goal is to navigate
through a maze to reach a target location. The ant needs to coordinate its four legs to
maintain balance while moving toward the goal. The maze has walls that the ant cannot pass
through. The task is considered successful when the ant reaches within 0.5 meters of the
target location.
"""

# ========== 领域提示 (关键差异) ==========
additional_prompt = """
Most Importantly: As for this task, the agent should firstly learn how to coordinate various
parts of the body itself before it finally reach the goal. Like a baby should learn how to walk
before finally walking to the goal. Therefore, when you design state representation and reward,
you should cosider how to make the agent learn to coordinate various parts of the body as well
as how to finally reach the goal.
"""

# 为什么需要这个提示？
"""
原因分析:
1. 复杂协调: 四足机器人需要协调 12 个关节
2. 分阶段学习:
   - 阶段 1: 学会协调四足，保持平衡 (不摔倒)
   - 阶段 2: 学会向目标移动 (导航)
3. 常见失败:
   - 直接奖励目标距离 → 摔倒 (还没学会走路就想跑)
   - 忽略身体协调 → 震荡、原地转圈
4. 关键成功因素:
   - 对角线步态 (四足动物的自然模式)
   - 身体稳定性 (保持 z 高度)
   - 平滑运动 (避免震荡)

这个提示引导 LLM:
- 先关注身体协调特征
- 再关注目标导向特征
- 设计分阶段的奖励
"""

```python
def post_writing_checklist(prompt_template):
    """
    提示词撰写后的检查清单
    """
    print("=== 提示词撰写后检查清单 ===")

    checks = []

    # 1. 必需组件
    checks.append(("包含任务描述", "task description" in prompt_template.lower()))
    checks.append(("包含状态语义", "s[0]" in prompt_template and "s[" in prompt_template))
    checks.append(("包含单位信息", "unit is" in prompt_template.lower()))
    checks.append(("包含维度约束", "dimensional" in prompt_template.lower()))
    checks.append(("包含输出范围", "[-100, 100]" in prompt_template))
    checks.append(("包含函数签名", "def revise_state(s)" in prompt_template))
    checks.append(("包含示例代码", "```python" in prompt_template))

    # 2. 任务特定组件
    has_additional = "Most Importantly" in prompt_template or "additional" in prompt_template.lower()
    checks.append(("包含领域提示 (如需要)", has_additional))

    has_lipschitz = "lipschitz" in prompt_template.lower()
    checks.append(("包含 Lipschitz 分析", has_lipschitz))

    has_analysis = "(a)" in prompt_template or "(b)" in prompt_template
    checks.append(("包含分析引导", has_analysis))

    # 总结
    pass_count = sum(1 for _, result in checks if result)
    total_count = len(checks)

    for check, result in checks:
        print(f"{'✅' if result else '❌'} {check}")

    print(f"\n通过率: {pass_count}/{total_count} ({pass_count/total_count*100:.1f}%)")

    if pass_count == total_count:
        print("🎉 提示词撰写完成，可以开始实验！")
        return True
    elif pass_count >= total_count * 0.8:
        print("⚠️  提示词基本完成，建议优化缺失项。")
        return False
    else:
        print("❌ 提示词不完整，需要大幅改进。")
        return False
