## 🎯 核心发现

### 1. 项目定位

这是 **LESR (LLM-Empowered State Representation)** 从机器人控制到金融 DRL 决策的迁移实践，但存在**思维转换不足**的根本问题。

### 2. 三大架构问题

```
┌─────────────────────────────────────────────────────────────┐
│  ❌ 问题 1: State Contract 不一致                             │
├─────────────────────────────────────────────────────────────┤
│  • Prompt 已说明 native contract                             │
│  • 但 CoT 反馈的 source_dim 仍用 schema.dim()               │
│  • 导致 Lipschitz 分析错位                                    │
│  • 影响: LLM 收到错误的维度反馈                              │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│  ❌ 问题 2: System Prompt 缺少 Native 绑定                   │
├─────────────────────────────────────────────────────────────┤
│  • trading_lesr_prior_v1 仍是通用叙述                        │
│  • Native state contract 没硬编码到 prompt                  │
│  • 依赖外部配置，容易回退                                    │
│  • 影响: 换配置时可能丢失 native 约束                        │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐

└─────────────────────────────────────────────────────────────┘
```

### 3. Intrinsic Reward 的核心问题

```
⚠️  致命问题: G2 Mode 的 Raw-State Fallback 不足

审计发现:
• TD3 最佳候选: intrinsic_probe_delta_sharpe = 0.0
• TD3 G2 行为几乎复制 G0
• TD3 G3 行为几乎复制 G1

结论: intrinsic_reward(s) 在原始状态上几乎没有信号！

根本原因:
1. Prompt 要求不够明确
2. LLM 倾向于让 intrinsic_reward 完全依赖扩展维度
3. 验证只检查"能运行"，没检查"raw-state 分支的有效性"

示例问题代码:
```python
# ❌ 问题: intrinsic_reward 在原始状态上几乎为常数
def revise_state(s):
    momentum = (s[close_idxs] - s[open_idxs]) / (s[open_idxs] + 1e-8)
    return np.concatenate([s, momentum])

def intrinsic_reward(updated_s):
    # 只使用扩展维度
    momentum = updated_s[-len(close_idxs):]
    return float(np.mean(momentum) * 10.0)

# 当调用 intrinsic_reward(raw_state) 时:
# • 没有扩展维度
# • 访问错误的索引
# • 返回值没有意义
```
```

### 4. 最关键的问题：思维转换不足

```
┌─────────────────────────────────────────────────────────────┐
│  🚨 根本问题: "机器人思维"未完全转换到"金融思维"             │
└─────────────────────────────────────────────────────────────┘

原始 LESR (机器人控制):
━━━━━━━━━━━━━━━━━━━━
• intrinsic_reward = 探索奖励
• 目标: 鼓励访问新状态
• 奖励: 多样性、新颖性

金融 DRL (应该这样):
━━━━━━━━━━━━━━━━━
• intrinsic_reward = 风险感知引导
• 目标: 引导风险-aware 的投资决策
• 奖励: 风险调整收益、多样化、风险惩罚

当前项目 (错误):
━━━━━━━━━━━━━━
• intrinsic_reward = 探索奖励 ❌
• 把金融交易当成机器人控制 ❌
• Lipschitz = 平滑性，而非风险-收益权衡稳定性 ❌

⚠️  这不是"换个 Prompt"能解决的问题，而是需要重新设计 Objective！
```

#### 2. 重新设计 Prompt: 从"探索奖励"到"风险感知引导"

```python
# ❌ 当前 (机器人思维)
intrinsic_reward(s) = exploration_bonus(s)
  • 鼓励访问新状态
  • 奖励多样性

# ✅ 应该 (金融思维)
intrinsic_reward(s) = risk_aware_guidance(s)
  • 惩罚过度风险暴露
  • 奖励风险调整后的收益
  • 引导多样化配置
  • 考虑交易成本
```

#### 3. 增强 Raw-State Fallback 验证

```python
def validate_intrinsic_reward_raw_fallback(code, raw_state_sample):
    """验证 intrinsic_reward 的 raw-state fallback"""

    # 1. 测试 raw-state 分支
    raw_rewards = [intrinsic_reward(s) for s in raw_states]

    # 2. 检查非平凡性
    if np.std(raw_rewards) < 0.01:
        return False, "raw-state intrinsic reward is nearly constant"

    # 3. 检查 action-sensitive
    # 修改持仓，reward 应该变化
    base_reward = intrinsic_reward(base_state)
    test_state = base_state.copy()
    test_state[holdings_idxs] *= 1.5
    test_reward = intrinsic_reward(test_state)

    if abs(test_reward - base_reward) < 0.01:
        return False, "intrinsic reward is action-insensitive"

    return True, "valid"
```

#### 5. 算法特定适配

```python
# 不同算法对 intrinsic reward 的敏感度不同:

TD3:
  - 对噪声敏感，需要平滑的 intrinsic 信号
  - 避免 hard thresholds
  - 使用 confidence-gated penalties

PPO:
  - 对 clipped reward 敏感
  - intrinsic 不能过大 (建议 [-10, 10])
  - 使用 probability-weighted signals

SAC:
  - 对 exploration bonus 敏感
  - intrinsic 应引导 WHAT to explore，而非鼓励探索
  - 使用 entropy-aware signals

A2C:
  - 对 variance 敏感
  - intrinsic 需要稳定 (低方差)
  - 使用 baseline-subtracted signals
```

## 🎓 关键洞察

### 把 LESR 从机器人搬到金融，需要重新设计 Objective！

```
不是简单的"换个 Prompt"，而是:

机器人控制:
━━━━━━━━━━
状态 → 物理特征 → 探索奖励 → 任务完成

金融 DRL:
━━━━━━━
状态 → 风险暴露 → 风险感知引导 → 风险调整收益

⚠️  这是本质差异，不能只靠 Prompt 调整解决！
```

### 具体改进的 Objective

```python
# ❌ 当前 (错误)
objective = "maximize cumulative return + exploration bonus"

# ✅ 应该 (正确)
objective = """
maximize risk-adjusted returns (Sharpe Ratio)
subject to:
  - position limits
  - turnover constraints
  - drawdown limits
  - transaction costs

using:
  - revise_state: expose risk exposures and portfolio constraints
  - intrinsic_reward: guide risk-aware decisions (NOT exploration!)
"""
```

### 示例：正确的金融 DRL Intrinsic Reward

```python
# ✅ 正确: 风险感知的 intrinsic_reward

def intrinsic_reward(s):
    """
    风险感知的内在奖励

    关键: 必须在原始状态上也有效！
    """
    # 1. 风险调整收益 (正项)
    risk_adjusted_return = trend_strength / (market_volatility + 1e-8)
    reward_risk_adjusted = 50.0 * np.tanh(risk_adjusted_return)

    # 2. 多样化奖励 (正项)
    diversification_score = (1.0 - concentration) * entropy
    reward_diversification = 20.0 * diversification_score

    # 3. 集中度惩罚 (负项)
    penalty_concentration = -30.0 * (concentration ** 2)

    # 4. 波动率惩罚 (负项)
    penalty_volatility = -20.0 * market_volatility

    # 5. 交易成本惩罚 (负项)
    penalty_turnover = -10.0 * turnover

    # 6. 市场状态交互
    if trend_strength > 0:  # 牛市
        reward_exposure = 10.0 * np.clip(exposure - 0.5, -0.5, 0.5)
    else:  # 熊市
        reward_exposure = -10.0 * exposure

    # 综合奖励
    total = (
        reward_risk_adjusted +
        reward_diversification -
        penalty_concentration -
        penalty_volatility -
        penalty_turnover +
        reward_exposure
    )

    return np.clip(total, -100, 100)
```
